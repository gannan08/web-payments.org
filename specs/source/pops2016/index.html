This specification describes a signature suite that includes the ability to cryptographically prove when the digital signature was created by anchoring the signature to a hash in a publically available merkle tree.

This is an experimental specification and is undergoing regular revisions. It is not fit for production deployment.

Introduction

This specification describes a signature suite created in 2016 that includes the ability to cryptographically prove when the digital signature was created by anchoring the signature to a hash in a publically available merkle tree. This signature suite extends the Linked Data Signatures [[LD-SIGNATURES]] specification. It uses the RDF Dataset Normalization Algorithm [[RDF-DATASET-NORMALIZATION]] to transform the input document into its normal form. It uses SHA-256 [[RFC6234]] as the message digest algorithm and RSASSA-PKCS1-v1_5 [[RFC3447]] as the signature algorithm. The major change addition to the signature mechanism is the inclusion of several Merkle tree hashes into the signature hash.

Terminology

The following terms are used to describe concepts involved in the generation and verification of the Linked Data Signature 2015 signature suite.

merkle tree
A tree in which every non-leaf node is labelled with the hash of the labels or values (in case of leaves) of its child nodes. Wikipedia has a more complete definition.
signature suite
A specified set of cryptographic primitives typically consisting of a normalization algorithm, a message digest algorithm, and a signature algorithm that are bundled together by cryptographers for developers for the purposes of safety and convenience.
normalization algorithm
An algorithm that takes an input document that has more than one possible representation and always transforms it into a normal form. This process is sometimes also called canonicalization.
message digest algorithm
An algorithm that takes an input message and produces a cryptographic output message that is often times many orders of magnitude smaller than the input message. These algorithms are often 1) very fast, 2) non-reversible, 3) change when even one bit of the input message changes, and 4) produce different outputs for different inputs.
signature algorithm
An algorithm that takes an input message and produces an output value where the receiver of the message can mathematically verify that the message came from a particular sender and has not been modified in transit.

The Proof of Publication 2016 Signature Suite

The Proof of Publication Signature 2016 signature suite MUST be used in conjunction with the signing and verification algorithms in the Linked Data Signatures [[LD-SIGNATURES]] specification. The suite consists of the following algorithms:

Parameter Value Specification
normalizationAlgorithm https://w3id.org/security#URDNA2015 [[RDF-DATASET-NORMALIZATION]]
digestAlgorithm http://example.com/digests#sha256 [[RFC6234]]
signatureAlgorithm http://www.w3.org/2000/09/xmldsig#rsa-sha1 [[RFC3447]]

Generating a Proof of Publication

A merkle tree-based proof of publication is generated by adding a value into a public merkle tree and then including that value in the digital signature. There are several values that MUST be included in order to ensure the proof is valid: the merkle root, the parent, the left sibling, and the right sibling.

Another alternative is to include the path from our entry back to the merkle root. We're trying to minimize the data that goes into the signature by making it extremely unlikely that the three hashes just happen to be clustered elsewhere in the same tree.

To generate the merklePublicationProof, perform the following steps:

  1. Add an entry to a public merkle tree. Generate hashes for the merkle root, the parent, the left sibling, and the right sibling using the SHA-256 message digest algorithm [[RFC6234]].
  2. Generate proof value by joining the following strings in order, using a colon : (U+003A) as the separator:
    1. pops2016
    2. the SHA-256 message digest of the merkle root
    3. the SHA-256 message digest of the parent
    4. the SHA-256 message digest of the left sibling
    5. the SHA-256 message digest of the right sibling
  3. Set the merklePublicationProof to proof value.

Modification to Algorithms

The following modifications must be made to the algorithms specified in the Linked Data Signatures [[LD-SIGNATURES]] specification.

Signature Algorithm

After performing step #4 and prior to performing step #5 of the Signature Algorithm:

  1. If a merklePublicationProof value exists, add an entry to headers with a key of https://w3id.org/security#merklePublicationProof and the associated value.
